### Dockerfile de ReferГӘncia

[Source: missГЈo SM вҖ” baseado em deployment-options.md e stack da Story 1.1]

```dockerfile
FROM node:20-slim

# System dependencies
RUN apt-get update && apt-get install -y \
    python3 \
    python3-pip \
    ffmpeg \
    curl \
    unzip \
    && rm -rf /var/lib/apt/lists/*

# yt-dlp
RUN pip3 install yt-dlp --break-system-packages

# Deno (obrigatГіrio para resolver assinaturas YouTube em 2026)
RUN curl -fsSL https://deno.land/install.sh | sh
ENV DENO_INSTALL="/root/.deno"
ENV PATH="$DENO_INSTALL/bin:$PATH"

# App
WORKDIR /app
COPY backend/ .
RUN npm install --production

# HuggingFace Spaces: porta obrigatГіria = 7860
ENV PORT=7860
ENV NODE_ENV=production
EXPOSE 7860

CMD ["node", "server.js"]
```

### Problema CrГӯtico: Bloqueio de IP do YouTube em 2026

[Source: docs/deployment-options.md вҖ” Alerta CrГӯtico]

O YouTube bloqueia IPs de datacenters (incluindo HuggingFace). Erro tГӯpico:

```
ERROR: Sign in to confirm you're not a bot.
Use --cookies-from-browser or --cookies for authentication.
```

**MitigaГ§ГЈo jГЎ implementada na Story 1.1:**
- `POST /api/cookies` aceita upload de `cookies.txt` do usuГЎrio
- O usuГЎrio exporta cookies do prГіprio browser com extensГЈo "Get cookies.txt locally"
- Backend usa `yt-dlp --cookies /caminho/cookies.txt URL` nas requests

**Deno вҖ” requisito obrigatГіrio 2026:**
yt-dlp requer Deno instalado para resolver assinaturas do YouTube. Sem Deno = erros 403.
O Dockerfile acima instala Deno e configura o PATH corretamente.
