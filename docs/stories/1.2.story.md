## Story

**As a** developer do Yutubo,
**I want** containerizar o backend com Docker e fazer deploy no HuggingFace Spaces,
**so that** a API fique acessível publicamente ($0, sem cartão) e o frontend Vercel possa consumi-la para realizar downloads reais.

## Acceptance Criteria

1. `Dockerfile` criado na raiz do repositório que constrói uma imagem com Node.js 20, yt-dlp (pip), ffmpeg (apt) e Deno — os mesmos binários que a Story 1.1 instalava manualmente na VM Oracle.
2. A imagem expõe e escuta na porta **7860** (requisito obrigatório do HuggingFace Spaces). A variável `PORT=7860` é definida via `ENV` no Dockerfile; `backend/lib/config.js` já lê `process.env.PORT` e portanto não precisa de alteração de código.
3. `README.md` na raiz do Space (arquivo `hf-space/README.md` no repositório, copiado como `README.md` no clone do Space) contém os metadados YAML obrigatórios do HuggingFace: `sdk: docker`, `title`, `emoji`, `colorFrom`, `colorTo`, `pinned`.
4. O Space HuggingFace `Juan-Felipe1/yutubo-backend` é criado, o Dockerfile é enviado via `git push` e o build conclui com sucesso (status "Running" no painel do Space).
5. `GET https://juan-felipe1-yutubo-backend.hf.space/api/health` retorna `{ "status": "ok", "version": "1.0.0" }` com HTTP 200.
6. `POST https://juan-felipe1-yutubo-backend.hf.space/api/analyze` com uma URL YouTube retorna metadata real (título, canal, duração, lista de qualidades) — confirma que yt-dlp + Deno estão funcionais dentro do container.
7. O frontend (`Descargador.dc.html` + `support.js`) é deployado no Vercel. A variável de ambiente `YUTUBO_API` (ou `window.YUTUBO_API` via `vercel.json` / script de inicialização) aponta para `https://juan-felipe1-yutubo-backend.hf.space`.
8. Teste end-to-end completo: abrir a URL Vercel do frontend, colar uma URL YouTube, analisar, selecionar MP4 1080p e iniciar download — o arquivo começa a ser baixado no browser.
9. CORS configurado no backend para aceitar requests do domínio Vercel do frontend (variável de ambiente `CORS_ORIGINS` definida como secret no Space ou via `ENV` no Dockerfile para a URL do Vercel).
10. O Space não acorda imediatamente após 48h de inatividade (sleep) — isso é comportamento esperado e documentado na story; o AC não exige uptime 24/7.

### Dockerfile de Referência

[Source: missão SM — baseado em deployment-options.md e stack da Story 1.1]

```dockerfile
FROM node:20-slim

# System dependencies
RUN apt-get update && apt-get install -y \
    python3 \
    python3-pip \
    ffmpeg \
    curl \
    unzip \
    && rm -rf /var/lib/apt/lists/*

# yt-dlp
RUN pip3 install yt-dlp --break-system-packages

# Deno (obrigatório para resolver assinaturas YouTube em 2026)
RUN curl -fsSL https://deno.land/install.sh | sh
ENV DENO_INSTALL="/root/.deno"
ENV PATH="$DENO_INSTALL/bin:$PATH"

# App
WORKDIR /app
COPY backend/ .
RUN npm install --production

# HuggingFace Spaces: porta obrigatória = 7860
ENV PORT=7860
ENV NODE_ENV=production
EXPOSE 7860

CMD ["node", "server.js"]
```

### Problema Crítico: Bloqueio de IP do YouTube em 2026

[Source: docs/deployment-options.md — Alerta Crítico]

O YouTube bloqueia IPs de datacenters (incluindo HuggingFace). Erro típico:

```
ERROR: Sign in to confirm you're not a bot.
Use --cookies-from-browser or --cookies for authentication.
```

**Mitigação já implementada na Story 1.1:**
- `POST /api/cookies` aceita upload de `cookies.txt` do usuário
- O usuário exporta cookies do próprio browser com extensão "Get cookies.txt locally"
- Backend usa `yt-dlp --cookies /caminho/cookies.txt URL` nas requests

**Deno — requisito obrigatório 2026:**
yt-dlp requer Deno instalado para resolver assinaturas do YouTube. Sem Deno = erros 403.
O Dockerfile acima instala Deno e configura o PATH corretamente.
